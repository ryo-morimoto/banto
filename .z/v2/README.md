# v2 具体化

## 目的

アーキテクチャ概観（`.z/curation/architecture-decision.md`）を実装可能な仕様に落とす。各サブディレクトリが具体化の 1 軸を担当する。ディレクトリ内のファイルは抽象度を段階的に下げ、最終的に設計判断なしで実装できるレベルまで到達させる。

## 運用

1. 各ディレクトリに README があり、スコープと成果物一覧を定義する
2. 成果物は個別の `.md` ファイルとして配置する
3. 「完了」= 追加の設計判断なしで実装に着手できる状態
4. ディレクトリ間の参照は相対パスで行う

## ディレクトリ

| ディレクトリ | 軸 | 答える問い |
|-------------|------|---------------------|
| `product/` | プロダクト要件 | 何を作るか？何を作らないか？ |
| `architecture/` | アーキテクチャ | コンポーネントはどう繋がるか？インターフェースは？ |
| `validation/` | 技術検証 | 未検証の仮定は何か？何を PoC するか？ |
| `plan/` | 実装計画 | どの順番で？何が何に依存するか？ |
| `operations/` | 運用設計 | どうデプロイし、監視し、更新するか？ |

## 上流

- 競合リサーチ: `.z/research/`
- 選択肢全景 & アーキテクチャ判断: `.z/curation/`
