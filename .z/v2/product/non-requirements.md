# v2 スコープ外

明示的に v2 では作らないもの。理由付き。

---

## 確定スコープ外（DRAFT.md で除外済み）

| 項目 | 理由 |
|------|------|
| マルチユーザー / チーム機能 | ターゲットは 1 人の開発者。認証・権限管理は不要 |
| クラウドホスティング (SaaS) | ローカルサーバー前提。Tailscale でリモートアクセス |
| ネイティブアプリ | PWA でカバー。ネイティブは将来検討 |

---

## v2 で意図的に後回しにするもの

| 項目 | 理由 | 将来の可能性 |
|------|------|------------|
| タスク依存関係 | コアループ（jot/throw/watch）の完成が先。依存関係は複雑度が高い | Phase 5+ |
| CI/CD 連携 | まずダッシュボードからの手動操作を完成させる | API/Webhook で後から対応可能 |
| Push 通知 | Web Push API は Phase 4。コアループに必須ではない | Phase 4 |
| 添付ファイル管理 | タスク説明にテキストで十分な初期段階 | Phase 5+ |
| タスクテンプレート / Playbook | Devin の Playbook は高評価だが、まず個別タスクの成功率を上げる | 実運用データが溜まってから |
| 長期分析 / ダッシュボード履歴 | リアルタイムビューが最優先。分析は後（marc-nuri の教訓） | データは event ledger に蓄積されるので後から実装可能 |
| エージェント間の自動連携 | multi-agent orchestration は複雑。1 タスク = 1 セッションの原則を守る | 需要と安定性を見て検討 |
| カスタム権限ポリシー | 「このツールは常に承認」等のルールベース自動承認。まず手動承認を確実に動かす | セッション内自動承認は v2 で対応（S7 チェックボックス） |
| ブラウザ操作の表示 | Devin の Chrome 操作のようなブラウザビュー。CLI エージェントにはないケイパビリティ | エージェント側の進化次第 |
