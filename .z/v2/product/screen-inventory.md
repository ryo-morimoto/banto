# 画面一覧

全画面/ビューの定義、表示要素、画面間遷移。

---

## 画面一覧

| ID | 画面名 | パス | 説明 |
|----|-------|------|------|
| S1 | ダッシュボード | `/` | メイン画面。全タスクをプロジェクト別に一覧表示 |
| S2 | タスク詳細 | `/tasks/:id` | タスクの詳細 + セッション履歴 + アクティブセッション |
| S3 | セッション詳細 | `/tasks/:taskId/sessions/:id` | セッションのターミナル or 構造化ビュー + タイムライン |
| S4 | タスク作成モーダル | (モーダル) | タスクのタイトル・説明・プロジェクト入力 |
| S5 | セッション開始モーダル | (モーダル) | エージェント選択 + 開始確認 |
| S6 | プロジェクト作成モーダル | (モーダル) | プロジェクト名・リポジトリパス入力 |
| S7 | 権限応答モーダル | (モーダル / インライン) | 権限リクエストの内容表示 + 承認/拒否 |
| S8 | 設定 | `/settings` | エージェント設定、通知設定 |

---

## S1: ダッシュボード

### レイアウト

```
┌──────────────────────────────────────────┐
│ banto                        [+ Task] [⚙] │
├──────────────────────────────────────────┤
│ 📌 Pinned                                │
│ ┌──────────────────────────────────────┐ │
│ │ Task A  ● running   CC  3m ago      │ │
│ │ Task B  ● permission  Codex         │ │
│ └──────────────────────────────────────┘ │
│                                          │
│ Project Alpha                            │
│ ┌──────────────────────────────────────┐ │
│ │ Task C  ● done   +42 -12  CC        │ │
│ │ Task D  ○ backlog                    │ │
│ └──────────────────────────────────────┘ │
│                                          │
│ Project Beta                             │
│ ┌──────────────────────────────────────┐ │
│ │ Task E  ● failed  ✕ exit 1  ACP     │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

### 表示要素（タスクカード）

| 要素 | 説明 |
|------|------|
| タイトル | タスク名 |
| ステータス | 最新セッションの状態（running / done / failed / waiting_permission / backlog） |
| エージェント | プロバイダー名（CC / Codex / OpenCode / ...） |
| 経過時間 | セッション開始からの経過 |
| Git サマリー | 完了時: +追加行 -削除行、変更ファイル数 |
| ピンアイコン | ピン留め状態 |

### 操作

- タスクカードをタップ → S2（タスク詳細）
- 「+ Task」ボタン → S4（タスク作成モーダル）
- 権限待ちタスクをタップ → S7（権限応答）
- 設定アイコン → S8

## S2: タスク詳細

### レイアウト

```
┌──────────────────────────────────────────┐
│ ← Task A                    [Start] [⋮]  │
├──────────────────────────────────────────┤
│ Description                              │
│ Fix auth bug in login flow               │
│                                          │
├──────────────────────────────────────────┤
│ Active Session                           │
│ ┌──────────────────────────────────────┐ │
│ │ Session #3  ● running  CC  5m       │ │
│ │ Tokens: 12k in / 8k out             │ │
│ │ [View Session]           [Stop]     │ │
│ └──────────────────────────────────────┘ │
│                                          │
├──────────────────────────────────────────┤
│ Session History                          │
│ #2  ● failed   3m   exit 1  2h ago     │
│ #1  ● done     8m   +42 -12  5h ago   │
└──────────────────────────────────────────┘
```

### 表示要素

| 要素 | 説明 |
|------|------|
| タスクタイトル・説明 | マークダウン表示 |
| アクティブセッション | 現在実行中のセッションカード（あれば 1 つ） |
| セッション履歴 | 過去のセッション一覧（新しい順） |
| 添付ファイル | アップロードされたファイル一覧 |

### 操作

- 「Start」→ S5（セッション開始モーダル）
- 「View Session」→ S3（セッション詳細）
- 「Stop」→ セッション停止（確認あり）
- 「Retry」（失敗セッション）→ 新セッション開始
- 「Resume」（resume 対応で中断済み）→ セッション再開
- タスク編集（タイトル・説明）
- ピン留めトグル
- ステータス変更

## S3: セッション詳細

プロバイダーの `terminal` capability で表示が分岐する。

### ターミナルモード（terminal: true）

```
┌──────────────────────────────────────────┐
│ ← Session #3          ● running  [Stop]  │
├──────────────────────────────────────────┤
│ ┌──────────────────────────────────────┐ │
│ │                                      │ │
│ │         Terminal Panel               │ │
│ │      (restty / xterm.js)             │ │
│ │                                      │ │
│ └──────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│ Status: running │ Tokens: 12k/8k │ 5m   │
├──────────────────────────────────────────┤
│ Timeline                                 │
│ 14:01 ● started                         │
│ 14:02 ▶ tool: Read(src/auth.ts)         │
│ 14:03 ▶ tool: Edit(src/auth.ts)         │
│ 14:05 ⚠ permission: Write(package.json) │
└──────────────────────────────────────────┘
```

### 構造化ビューモード（terminal: false）

```
┌──────────────────────────────────────────┐
│ ← Session #3          ● running  [Stop]  │
├──────────────────────────────────────────┤
│ Conversation                             │
│ ┌──────────────────────────────────────┐ │
│ │ 🤖 Reading src/auth.ts...           │ │
│ │ 📄 src/auth.ts (lines 1-50)         │ │
│ │ 🤖 Found the bug. Fixing...         │ │
│ │ ✏️ Edit src/auth.ts (+5 -3)          │ │
│ │ 🤖 Running tests...                 │ │
│ │ ▶ bash: bun test                    │ │
│ └──────────────────────────────────────┘ │
├──────────────────────────────────────────┤
│ Status: running │ Tokens: 12k/8k │ 5m   │
├──────────────────────────────────────────┤
│ Timeline                                 │
│ (same as terminal mode)                  │
└──────────────────────────────────────────┘
```

### 共通表示要素

| 要素 | 説明 |
|------|------|
| ステータスバー | 状態、トークン数、経過時間、コスト |
| タイムライン | イベント時系列（ツール使用、メッセージ、権限、状態変化） |
| Git サマリー | 完了時: 変更ファイル一覧、diff 統計 |

### 操作

- Stop → セッション停止
- ターミナルモード: キーボード入力でエージェントへ送信
- 権限リクエスト → S7（権限応答 UI）
- diff 表示切替（完了後）

## S4: タスク作成モーダル

### フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| プロジェクト | ✓ | ドロップダウン（既存プロジェクト一覧） |
| タイトル | ✓ | テキスト入力 |
| 説明 | - | マークダウン対応テキストエリア |
| 添付ファイル | - | ファイルアップロード |

---

## S5: セッション開始モーダル

### フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| エージェント | ✓ | ドロップダウン（利用可能なプロバイダー一覧） |

プロンプトはタスクのタイトル + 説明から自動構成。

---

## S6: プロジェクト作成モーダル

### フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| 名前 | ✓ | プロジェクト表示名 |
| リポジトリパス | ✓ | ローカルパス（バリデーション: 存在確認 + .git 確認） |

---

## S7: 権限応答 UI

### 表示要素

| 要素 | 説明 |
|------|------|
| ツール名 | 何を実行しようとしているか |
| 引数 | 構造化表示（raw JSON ではない） |
| コンテキスト | なぜこの操作が必要か（エージェントの説明、あれば） |

### 操作

- 承認（Approve）
- 拒否（Deny）
- 「このセッション中は同じ操作を自動承認」チェックボックス

---

## S8: 設定

### セクション

| セクション | 内容 |
|-----------|------|
| エージェント | 利用可能なプロバイダー一覧、パス設定、デフォルトエージェント |
| 通知 | Push 通知の ON/OFF、通知対象イベント |
| 表示 | テーマ、デフォルトフィルター |

---

## 画面遷移図

```
S1 ダッシュボード
├── タスクカードタップ → S2 タスク詳細
│   ├── 「Start」→ S5 セッション開始モーダル → S3 セッション詳細
│   ├── 「View Session」→ S3 セッション詳細
│   │   └── 権限リクエスト → S7 権限応答 UI
│   ├── 「Retry」→ S5 → S3
│   └── 「Resume」→ S3
├── 「+ Task」→ S4 タスク作成モーダル → S2
├── 権限待ちタスクタップ → S2 → S7
└── 設定アイコン → S8 設定
```
