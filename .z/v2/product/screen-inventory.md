# 画面一覧

全画面/ビューの定義、表示要素、画面間遷移。
競合 UI リサーチ（cmux, Happy Coder, Marc Nuri, vde-monitor, Devin, Superset, AgentOS, Amp 等）の優れたパターンを反映。

---

## 設計原則（競合リサーチから）

| 原則 | 根拠 |
|------|------|
| カードに 5-7 フィールド表示、ドリルダウン不要で判断可能 | 全成功ツールに共通。marc-nuri, cmux, happy-coder |
| 「要対応」が最上部 | 権限待ち/失敗が埋もれると放置される。cmux notification rings |
| ステータスは色で、詳細はテキストで | 色ドット（green/orange/red）で瞬間認識、テキストラベルで確認 |
| モバイルは「監視と承認」、フル操作はデスクトップ | ターミナル操作はスマホで非現実的。happy-coder, agentos の教訓 |
| 実行画面は provider.mode で分岐 | mode: "terminal" → ターミナル、"structured" → 構造化会話。architecture-decision D2 |
| コンテキスト使用率をカードに表示 | 「次にどこを見るべきか」の最良予測指標。marc-nuri |
| 2 モード設計（board + workspace）を避ける | Vibe Kanban, Nimbalyst で失敗。展開/ドリルダウンで代替。ui-ux-design-patterns |
| タスク = ユーザーの操作単位 | ユーザーの関心は「目的が達成されたか」。実行回数は内部概念。5why 分析 |

| ID | 画面名 | パス | 説明 |
|----|-------|------|------|

### 表示セクション（優先度順）

| 順位 | セクション | 表示条件 | 根拠 |
|------|----------|---------|------|
| 1 | Needs Attention | waiting_permission or failed のタスクがある | TR2: 見落とし防止。cmux notification rings |
| 2 | Pinned | ピン留めタスクがある | PM2 |
| 3 | プロジェクト別グルーピング | 常時 | PM1。完了フィルター時は done を非表示 |

### タスクカード要素

| 要素 | 表示位置 | 例 | 表示条件 |
|------|---------|-----|---------|
| ステータスドット | 行頭 | (green)(orange)(red)(check)(o) | 常時 |
| タスクタイトル | 1 行目左 | "Fix auth bug" | 常時 |
| エージェント名 | 1 行目右 | CC / Codex / ACP | 実行中 or 完了 |
| 経過時間 | 1 行目右 | 12m | running / waiting |
| ステータスラベル | 1 行目右端 | running / waiting_perm / failed / done | 常時 |
| プロジェクト名 | 2 行目（PJ グループ外のみ） | project-alpha | Pinned / Attention セクション |
| ブランチ名 | 2 行目 | feat/auth | 実行中 or 完了 |
| コンテキスト使用率 | 2 行目 | ctx 78% | running。Marc Nuri: "best predictor of where to look next" |
| diff サマリー | 2 行目 | +42 -12, 3 files | done |
| 権限リクエスト概要 | 2 行目 | "Write package.json" | waiting_permission |
| エラー概要 | 2 行目 | exit 1 | failed |
| 完了からの経過 | 2 行目 | 2h ago | done |

**根拠**: Marc Nuri（project + branch + model + context % + status + PR）、cmux（branch + ports + PR + notification text）、Happy Coder（status dot + model + tokens + context %）。全ツールで 5-7 フィールドがカードレベルに表示されている。

### モバイル固有のインライン操作

| 操作 | 表示条件 | 根拠 |
|------|---------|------|
| [Approve] [Deny] ボタン | waiting_permission | モバイルでの承認がメインユースケース（TR4, happy-coder） |
| [Retry] ボタン | failed | 説明修正 → リトライをモバイルでも可能に |

### ヘッダー要素

| 要素 | 説明 |
|------|------|
| 「+ Task」 | タスク作成モーダルを開く |
| 通知バッジ | 未読通知数 |
| 設定アイコン | 設定画面へ |

| +----------------------------------------------------------+ |
| | (orange)  CC   12m   waiting_permission                  | |

| | Latest: !! Permission: Write(package.json)               | |
| |                                                          | |
| | [Open]    [Approve]  [Deny]    [Stop]                    | |
| +----------------------------------------------------------+ |
|                                                              |
| History                                          [show all]  |
| +----------------------------------------------------------+ |
| | #2  (red) failed   CC    3m    exit 1       2h ago       | |

| +----------------------------------------------------------+ |

```

### 現在の実行状態

| 要素 | 説明 |
|------|------|
| ステータスドット + ステータス | waiting_permission |
| エージェント + 経過時間 | CC . 12m |
| ブランチ | feat/auth |
| トークン + コスト推計 | 14k in / 9k out . ~$0.12 |
| 最新イベント | 直近の重要イベント 1 行表示（MO6: 今何してるか） |
| アクションボタン | Open / Approve・Deny / Stop |

### 実行履歴

試行回数が見える折りたたみ可能な履歴。「セッション」という語は表示しない。

| 要素 | 説明 |
|------|------|
| 試行番号 | #2, #1（新しい順） |
| ステータス | (red) failed / (check) done |
| エージェント + 所要時間 | CC . 3m |
| 結果サマリー | exit 1（failed）/ +42 -12（done） |
| 完了からの経過 | 2h ago |

### 操作

| 操作 | 条件 |
|------|------|
| > Start | 実行中でない |
| Open → S3 | 実行中 |
| Approve / Deny | waiting_permission |
| Stop | running / waiting_permission |
| Retry（履歴の項目） | failed |
| Resume（履歴の項目） | failed + resume 対応エージェント |
| pin ピントグル | 常時 |
| .. メニュー | 編集、ステータス変更、削除 |

## S3: 実行ビュー

実行中のタスクを「開いた」ときの画面。`provider.mode` で **2 つのモード**に分岐する。共通部分（ステータスバー + タイムライン）は同じ。

### 共通ヘッダー + ステータスバー

```
+-------------------------------------------------------------+
| <- Fix auth bug                     (orange) waiting  [Stop] |
+-------------------------------------------------------------+
| CC . feat/auth . 12m . 14k/9k tokens . ~$0.12 . ctx 78%    |
+-------------------------------------------------------------+
```

ステータスバー要素: エージェント名 / ブランチ / 経過時間 / トークン / コスト推計 / ctx %

### モード A: ターミナルビュー（mode: "terminal"）

Claude Code (PTY), PTY fallback 用。

```
+-------------------------------------------------------------+
| [status bar]                                                 |
+-------------------------------------------------------------+
| +----------------------------------------------------------+|
| |                                                          ||
| |              Terminal Panel                              ||
| |           (restty / xterm.js)                            ||
| |                                                          ||
| |  $ claude ----------------------------------------       ||
| |  Reading src/auth.ts...                                  ||
| |  Found the issue on line 42.                             ||
| |  Editing src/auth.ts...                                  ||
| |  +----------------------------------------------+       ||
| |  | !! Permission: Write package.json             |       ||
| |  | [Approve in dashboard]                        |       ||
| |  +----------------------------------------------+       ||
| |                                                          ||
| +----------------------------------------------------------+|
+-------------------------------------------------------------+
| Timeline (collapsible)                                       |
| 14:01  * started                                            |
| 14:02  > Read(src/auth.ts)                                  |
| 14:03  @ Edit(src/auth.ts) +5 -3                            |
| 14:05  !! Permission: Write(package.json)  [Approve]         |
+-------------------------------------------------------------+
```

ターミナルパネル:
- フル PTY 出力のリアルタイム描画
- キーボード入力でエージェントへ送信可能（IN2: mid-session control）
- スクロールバック対応（ring buffer → 再接続で replay）

### モード B: 構造化ビュー（mode: "structured"）

Codex (app-server), ACP 用。

```
+-------------------------------------------------------------+
| [status bar]                                                 |
+-------------------------------------------------------------+
| Conversation                                                 |
| +----------------------------------------------------------+|
| | [bot] Reading src/auth.ts to understand the issue...     ||
| |                                                          ||
| | [file] src/auth.ts (lines 35-50)                         ||
| | +----------------------------------------------+         ||
| | | 35: const redirect = req.query.next          |         ||
| | | 36: if (!redirect) {                         |         ||
| | | ...                                          |         ||
| | +----------------------------------------------+         ||
| |                                                          ||
| | [bot] Found the bug. The redirect URL is not being       ||
| |    validated. Fixing now...                              ||
| |                                                          ||
| | [edit] Edit src/auth.ts (+5 -3)                          ||
| | +----------------------------------------------+         ||
| | | - const redirect = req.query.next            |         ||
| | | + const redirect = validateUrl(...)          |         ||
| | +----------------------------------------------+         ||
| |                                                          ||
| | [bot] Running tests to verify the fix...                 ||
| | > bash: bun test auth                                    ||
| | +----------------------------------------------+         ||
| | | v 12 tests passed                            |         ||
| | +----------------------------------------------+         ||
| +----------------------------------------------------------+|
|                                                              |
| [Message input]                            [Send]            |
+-------------------------------------------------------------+
| Timeline (collapsible, same as Mode A)                       |
+-------------------------------------------------------------+
```

構造化ビュー:
- エージェントのメッセージ、ファイル読み込み、編集、コマンド実行を型付きで表示
- ファイル読み込み → コードブロック、編集 → inline diff、コマンド → 出力ブロック
- メッセージ入力欄で mid-session control（IN2: sendMessage()）
- Happy Coder の教訓: raw JSON を表示しない。構造化した視覚表現にする

### 共通: タイムライン

両モード共通で画面下部に表示（折りたたみ可能）。

| イベント種別 | 表示形式 | 例 |
|------------|---------|-----|
| started | * + 時刻 | 14:01 * started |
| tool_use (Read) | > + ファイルパス | 14:02 > Read(src/auth.ts) |
| tool_use (Edit) | @ + ファイルパス + diff stats | 14:03 @ Edit(src/auth.ts) +5 -3 |
| tool_use (Bash) | > + コマンド | 14:04 > bash: bun test |
| permission_request | !! + ツール + 対象 + [Approve] | 14:05 !! Permission: Write(package.json) |
| status_changed | * + 新ステータス | 14:06 * done (exit 0) |
| error | x + メッセージ | 14:06 x Process exited with code 1 |
| cost_update | $ + トークン | 14:06 $ 14k in / 9k out |

### 完了後の追加表示

```
+-------------------------------------------------------------+

## S4: タスク作成モーダル

| フィールド | 必須 | 説明 |
|-----------|------|------|
| プロジェクト | yes | ドロップダウン（既存一覧）+ 「新規作成」リンク |
| タイトル | yes | テキスト入力 |
| 説明 | - | マークダウン対応テキストエリア |

## S5: 実行開始モーダル

| フィールド | 必須 | 説明 |
|-----------|------|------|
| エージェント | yes | 利用可能なプロバイダー一覧。各エージェントの mode + capability アイコン付き |

プロンプトはタスクのタイトル + 説明から自動構成。エージェント一覧に表示:
- mode: `terminal` | `structured`
- resume: `true` | `false`
- permissions: `true` | `false`

## S7: 権限応答 UI

Happy Coder の最大の失敗（raw JSON 表示、ボタン無反応）を踏まえた設計。

### レイアウト

```
+-------------------------------------------+
| !! Permission Request                      |
|                                            |
| Fix auth bug                               |
|                                            |
| Tool:   Write                              |
| File:   package.json                       |
| Action: Add "zod" to dependencies          |
|                                            |
| +----------------------------------------+ |
| | + "zod": "^3.22.0"                    | |
| +----------------------------------------+ |
|                                            |
| [ ] Remember for this task                 |
|                                            |
| [Deny]                      [Approve]      |
+-------------------------------------------+
```

### 表示要素

| 要素 | 説明 | 根拠 |
|------|------|------|
| タスク名 | どのタスクの権限か | コンテキスト提供 |
| ツール名 | 何を実行するか | IN1: 構造化表示 |
| 対象ファイル | どのファイルに | IN1 |
| アクション説明 | なぜ（エージェント側の説明、あれば） | IN1 |
| diff プレビュー | 変更内容（Write の場合） | 判断材料 |
| Remember チェック | 同タスク内の現在の実行で同種操作を自動承認 | IN4 |

## レスポンシブ戦略

| ブレークポイント | レイアウト変更 | 根拠 |
|---------------|-------------|------|
| デスクトップ (>=1024px) | フルレイアウト。実行ビューはターミナル + タイムライン並列 | メイン作業環境 |
| タブレット (768-1023px) | タイムラインを折りたたみ。カード 2 列 | サブモニター用途 |
| モバイル (< 768px) | シングルカラム。Needs Attention にインラインアクション。ターミナルは簡易表示 | 朝チェック・承認専用 |

モバイルでのターミナル操作は非推奨（画面サイズ + キーボード問題）。
AgentOS Issue #20（Chrome Android でキーボード開くと UI 消失）、vde-monitor Issue #16（スクロール不可）の轍を踏まない。
