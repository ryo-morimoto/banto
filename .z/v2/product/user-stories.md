# ユーザーストーリー

コアループ（Jot / Throw / Watch）+ 横断的な関心事で分類。

---

## Jot（タスクを書く）

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| J1 | タスクを作成したい | WHEN タイトルを入力して保存 THEN タスクが backlog に追加される |
| J2 | タスクに説明を書きたい | WHEN 説明欄にマークダウンを入力 THEN 保存・表示される |
| J3 | タスクをプロジェクトに紐付けたい | WHEN プロジェクトを選択して保存 THEN そのプロジェクト配下に表示される |
| J4 | タスクをピン留めしたい | WHEN ピンをトグル THEN ダッシュボード上部に固定表示される |
| J5 | タスクのステータスを手動変更したい | WHEN ステータスを変更 THEN backlog / active / done が切り替わる |
| J6 | プロジェクトを登録したい | WHEN リポジトリパスと名前を入力 THEN プロジェクトが作成される |
| J7 | タスクに添付ファイルを付けたい | WHEN ファイルをアップロード THEN タスク詳細から参照できる |

---

## Throw（エージェントに投げる）

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| T1 | タスクからセッションを開始したい | WHEN タスク詳細で「Start」を押す THEN エージェントがプロンプトを受けて実行開始する |
| T2 | 使用するエージェントを選択したい | WHEN セッション開始時にエージェントを選ぶ THEN 選択したエージェントのプロバイダーで起動する |
| T3 | セッションを停止したい | WHEN 「Stop」を押す THEN エージェントプロセスが停止し、ステータスが更新される |
| T4 | 失敗したセッションをリトライしたい | WHEN 失敗セッションで「Retry」を押す THEN 新しいセッションが同じプロンプトで開始される |
| T5 | セッションを再開したい | WHEN resume 対応エージェントで「Resume」を押す THEN 中断した地点から続行される |

---

## Watch（結果を見る）

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| W1 | 全タスクの状態を一目で把握したい | WHEN ダッシュボードを開く THEN 全プロジェクトのアクティブタスクがステータス付きで表示される |
| W2 | セッションのターミナル出力を見たい | WHEN terminal 対応プロバイダーのセッションを開く THEN ターミナルパネルにリアルタイム出力が表示される |
| W3 | セッションの構造化イベントを見たい | WHEN セッション詳細を開く THEN タイムラインにイベント（ツール使用、メッセージ、状態変化）が時系列で表示される |
| W4 | セッションの Git 差分を確認したい | WHEN セッション完了後に diff を見る THEN 変更ファイル一覧と差分が表示される |
| W5 | 権限リクエストに応答したい | WHEN エージェントが権限を要求 THEN ダッシュボードに構造化された承認/拒否 UI が表示される |
| W6 | 通知を受け取りたい | WHEN セッションが完了/失敗/権限待ちになる THEN Push 通知が届く |
| W7 | プロトコルモードの会話を閲覧したい | WHEN terminal: false のプロバイダーのセッションを開く THEN メッセージとツール呼び出しが構造化ビューで表示される |
| W8 | セッションのコスト（トークン数）を確認したい | WHEN セッション詳細を開く THEN 入力/出力トークン数とコストが表示される |

## 横断: プロジェクト管理

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| P1 | プロジェクト別にタスクを一覧したい | WHEN ダッシュボードを見る THEN タスクがプロジェクトごとにグルーピングされている |
| P2 | ピン留めタスクを最上部に表示したい | WHEN ピン留めタスクがある THEN プロジェクトグループより上に表示される |
| P3 | 完了タスクを非表示にしたい | WHEN フィルターで done を除外 THEN 完了タスクが一覧から消える |

---

## 横断: リアルタイム性

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| R1 | ページリロードなしで状態更新を受け取りたい | WHEN セッション状態が変わる THEN ダッシュボードがリアルタイムに反映される |
| R2 | 別デバイスから同じ状態を見たい | WHEN スマホでダッシュボードを開く THEN PC と同じ情報が表示される |
| R3 | ターミナル接続が切れても再接続したい | WHEN WebSocket が切断して再接続 THEN スクロールバックから状態が復元される |

---

## 横断: 耐障害性

| ID | ストーリー | 受け入れ基準 |
|----|----------|------------|
| F1 | サーバー再起動後に孤立セッションを検知したい | WHEN サーバーが再起動する THEN 実行中だったセッションが検知・通知される |
| F2 | resume 可能なエージェントは自動再開したい | WHEN resume 対応エージェントの孤立セッションがある THEN 自動的に再開が試みられる |
